# 专业表与学院表设计

## 3.X.1 学院表 (college)

学院表用于存储学校下设各学院的基础信息，为教师所属院系、专业归属等提供标准化的学院数据字典，避免各业务表中重复存储院系名称，保证数据的一致性与可维护性，包含学院ID、学院名称、学院编号、院长及描述等属性。其中 id 为自增主键，college_name 存储学院全称用于前台展示，college_code 为唯一约束用于系统内部标识学院，dean 记录学院负责人。其表结构如表3.X所示。

表3.X 学院表

| 字段名 | 字段类型及长度 | 是否为空 | 是否为主键 | 字段说明 |
|--------|--------------|---------|-----------|---------|
| id | BIGINT | 否 | 是 | 学院ID，自增 |
| college_name | VARCHAR(50) | 否 | 否 | 学院名称，唯一 |
| college_code | VARCHAR(20) | 否 | 否 | 学院编号，唯一 |
| dean | VARCHAR(30) | 是 | 否 | 院长姓名 |
| description | VARCHAR(200) | 是 | 否 | 学院简介 |
| created_at | DATETIME | 否 | 否 | 创建时间 |
| updated_at | DATETIME | 否 | 否 | 更新时间 |

## 3.X.2 专业表 (major)

专业表用于存储各学院开设的专业信息，为学生选择专业方向、课程归属及排课约束提供结构化的专业数据支持，替换原有学生表中直接存储专业名称字符串的方式，实现专业数据的规范化管理，包含专业ID、专业名称、专业编号、所属学院ID、学制等属性。其中 id 为自增主键，major_code 为唯一约束用于系统内部标识专业编号，college_id 作为外键关联学院表，确定专业与学院的归属关系，duration 记录该专业的标准学制年限。其表结构如表3.X所示。

表3.X 专业表

| 字段名 | 字段类型及长度 | 是否为空 | 是否为主键 | 字段说明 |
|--------|--------------|---------|-----------|---------|
| id | BIGINT | 否 | 是 | 专业ID，自增 |
| major_name | VARCHAR(50) | 否 | 否 | 专业名称 |
| major_code | VARCHAR(20) | 否 | 否 | 专业编号，唯一 |
| college_id | BIGINT | 否 | 否 | 所属学院ID，外键 |
| duration | INT | 是 | 否 | 学制年限 |
| description | VARCHAR(200) | 是 | 否 | 专业简介 |
| created_at | DATETIME | 否 | 否 | 创建时间 |
| updated_at | DATETIME | 否 | 否 | 更新时间 |

## 与现有系统的关联

新增这两个表后，Student 表中原有的 `major` 字段由直接存储专业名称字符串改为通过 major_id 外键关联专业表，Teacher 表中原有院系字段改为通过 college_id 关联学院表，消除数据冗余，实现字典统一管理。实体关系为：**学院(1) → 专业(N) → 学生(N)**，**学院(1) → 教师(N)**。

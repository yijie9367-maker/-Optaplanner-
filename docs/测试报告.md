## 三、测试中发现的问题及修正

### 问题1：Emoji字符编码不一致
- **现象**：`contains("👋")` 断言失败
- **原因**：Java源文件emoji在Windows GBK编译下与运行时字符串不匹配
- **修正**：改为关键词+长度组合断言

### 问题2：ChatbotService.java 结构损坏
- **现象**：mvn compile 失败，第158行多余 `}`
- **原因**：sed删除流式代码时残留闭合花括号
- **修正**：删除多余 `}` 恢复类结构

### 问题3：H2数据库不兼容MySQL语法
- **现象**：@DataJpaTest 运行时报 `CREATE DATABASE ... CHARACTER SET utf8mb4` 语法错误
- **原因**：项目 schema.sql 使用了MySQL专有语法，H2无法解析
- **修正**：添加 `test/resources/application.properties` 禁用测试环境schema初始化；Repository测试需MySQL环境，留到集成测试阶段

### 问题4：未登录接口状态码判断偏差
- **现象**：期望 `403 Forbidden`，实际返回 `401 Unauthorized`
- **原因**：Spring Security对未认证请求返回401（无凭证），403用于已认证但权限不足
- **修正**：改为 `is4xxClientError()` 兼容两种状态码

### 问题5：pom.xml 缺少测试依赖
- **现象**：MockMvc和@DataJpaTest编译失败，找不到类
- **原因**：项目未引入 `spring-security-test` 和 `h2`
- **修正**：在 pom.xml 中添加两项 test scope 依赖

## 3.4.1 数据库选型

本系统选用 **MySQL 8.4** 关系型数据库作为持久化存储方案，连接驱动为 MySQL Connector/J 8.0.33，并通过 Spring Data JPA（Hibernate）实现对象关系映射。选型理由如下：

**1. 关系型数据天然适配排课业务。** 排课系统的核心实体——班级（class_group）、学生（student）、教师（teacher）、课程（course）、教室（classroom）——之间存在明确的外键关联关系。例如排课记录（schedule）需同时引用课程、班级、教室与教师四张表，考试安排（exam）需关联课程、班级与教师，考试成绩（exam_grade）以联合唯一约束 `(exam_id, student_id)` 保证数据完整性。关系模型通过外键约束、1:N 和 M:N 关联天然表达排课场景的复杂数据结构，便于构建 JOIN 查询生成多维度课表视图。

**2. 事务机制保障排课结果的原子性。** 一键智能排课在执行过程中需批量生成数十条甚至上百条排课记录（schedule 行）。MySQL InnoDB 存储引擎提供 ACID 事务支持，确保排课结果的写入要么全部成功、要么全部回滚，避免出现部分时间段已排课、部分未排课的数据不一致状态。

**3. Spring Boot 生态深度集成。** 项目基于 Spring Boot 2.7 + Spring Data JPA 技术栈，Hibernate 自动根据 JPA 实体注解（`@Entity`、`@Column`、`@Table`）完成表结构映射。配置文件 `application.yml` 中设置 `ddl-auto: update`，在开发阶段可实现实体变更自动同步至数据库表结构，运维阶段切换为 `validate` 或 `none` 以保障生产安全。

**4. 字符集与国际化支持。** 连接字符串显式指定 `characterEncoding=UTF-8`，数据库启动命令配置 `character-set-server=utf8mb4` 与 `collation-server=utf8mb4_unicode_ci`，完整支持中文存储与排序，为课程名称、教师姓名、教室编号等中文字段提供可靠的字符集环境。

**5. Docker 容器化一键部署。** `docker-compose.yml` 编排 MySQL 8.4、Spring Boot 后端与 Vue 前端三个服务容器：MySQL 配置 `healthcheck` 心跳检测，后端通过 `depends_on` + `condition: service_healthy` 确保数据库就绪后才启动；连接信息以环境变量注入（`DB_URL`、`DB_USERNAME`、`DB_PASSWORD`），实现配置与代码分离，适配开发、测试、生产环境灵活切换。

**6. 连接池与性能。** Spring Boot 2.7 默认集成 HikariCP 连接池，自动管理数据库连接的创建、复用与释放，避免频繁建立 TCP 连接的开销。连接 URL 附加参数 `useSSL=false`（容器内网通信无需 TLS 加密层）、`serverTimezone=Asia/Shanghai`（统一时区）与 `allowPublicKeyRetrieval=true`（允许 JDBC 客户端获取 RSA 公钥用于密码加密传输），兼顾性能与安全。

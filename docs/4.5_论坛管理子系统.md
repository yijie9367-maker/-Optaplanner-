## 4.5 论坛管理子系统详细设计与实现

考试管理解决了教学评价的信息化，而师生之间围绕课程内容的日常交流同样需要一个开放的平台。论坛管理子系统以"发帖—评论—点赞/踩"为核心交互模式，为学生、教师和管理员提供一个课程相关的异步交流空间。管理员拥有置顶帖子、编辑与删除任意帖子及评论的权限；学生与教师可自由发帖、评论并对帖子进行点赞或点踩。本节分别阐述帖子管理与评论管理两个子模块的设计与实现。

### 4.5.1 论坛帖子管理模块

论坛帖子管理模块提供帖子的发布、查询、编辑、删除及置顶功能。帖子列表按时间排序，置顶帖始终显示在列表最前。每位用户可对帖子进行点赞或点踩操作，系统通过投票表记录用户的操作状态，支持首次投票、取消投票及切换投票类型三种行为。

**1. 实体层——ForumPost 实体**

```java
// ForumPost.java - 论坛帖子实体，映射数据库 forum_post 表
@Entity
@Table(name = "forum_post")
public class ForumPost {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private String title;             // 帖子标题
    @Column(columnDefinition = "TEXT")
    private String content;           // 帖子内容（长文本）
    private Long authorId;            // 发帖者ID
    private String authorName;        // 冗余存储，避免联表查询
    private String authorRole;        // 发帖者角色：student / teacher / admin
    private int likeCount = 0;        // 累计点赞数
    private int dislikeCount = 0;     // 累计点踩数
    private boolean isPinned = false; // 是否置顶，默认否
    private LocalDateTime createdAt;  // 创建时间，@PrePersist 自动填充
    private LocalDateTime updatedAt;  // 更新时间，@PreUpdate 自动填充
    // getter / setter 略...
}
```

帖子投票状态由专用的 ForumVote 实体记录，以帖子 ID、用户 ID 与用户角色三元组建立唯一约束，确保每位用户对每个帖子最多存在一条投票记录。服务层通过三段式逻辑处理投票：先查询已有记录，同类型则取消并减一，异类型则切换并同时调整两种计数，无记录则新增。

**2. 服务层——帖子列表与投票**

```java
// ForumService.java - 帖子列表查询与投票方法
@Service
public class ForumService {
    @Autowired
    private ForumPostRepository postRepository;   // 帖子 JPA 数据访问接口
    @Autowired
    private ForumVoteRepository voteRepository;   // 投票记录数据访问接口

    // 帖子列表：置顶帖优先，支持关键词搜索和时间排序
    public Page<ForumPostVO> getPosts(String keyword, String sort, int page, int size,
                                      Long userId, String userRole) {
        Sort.Direction dir = "oldest".equals(sort) ? Sort.Direction.ASC : Sort.Direction.DESC;
        Pageable pageable = PageRequest.of(page, size, Sort.by(dir, "createdAt"));

        if (keyword != null && !keyword.isBlank()) {
            // ① 关键词搜索路径：按标题模糊匹配分页返回
            return postRepository.findAllByTitleContainingIgnoreCase(keyword.trim(), pageable)
                    .map(p -> toVO(p, userId, userRole, false));
        }
        // ② 默认展示路径：置顶帖 + 普通帖合并
        List<ForumPost> pinned = postRepository.findByIsPinnedTrue();
        Page<ForumPost> normalPage = postRepository.findAll(pageable);
        if (page == 0) {
            // 首页：置顶帖置顶，去重后拼接普通帖
            List<ForumPost> merged = new ArrayList<>(pinned);
            normalPage.getContent().stream()
                    .filter(p -> merged.stream().noneMatch(x -> x.getId().equals(p.getId())))
                    .forEach(merged::add);
            return new PageImpl<>(
                    merged.stream().map(p -> toVO(p, userId, userRole, false)).collect(Collectors.toList()),
                    pageable, normalPage.getTotalElements() + pinned.size());
        }
        return normalPage.map(p -> toVO(p, userId, userRole, false));
    }

    // 点赞/踩 Toggle：同类型取消，异类型切换，无记录新增
    public ForumPostVO vote(Long postId, Long userId, String userRole, String voteType) {
        ForumPost post = postRepository.findById(postId)
                .orElseThrow(() -> new RuntimeException("帖子不存在"));
        Optional<ForumVote> existing = voteRepository
                .findByPostIdAndUserIdAndUserRole(postId, userId, userRole);
        if (existing.isPresent()) {
            ForumVote ev = existing.get();
            if (ev.getVoteType().equals(voteType)) {
                // ① 同类型：取消投票
                voteRepository.delete(ev);
                if ("like".equals(voteType)) post.setLikeCount(Math.max(0, post.getLikeCount() - 1));
                else post.setDislikeCount(Math.max(0, post.getDislikeCount() - 1));
            } else {
                // ② 异类型：切换投票
                if ("like".equals(voteType)) { post.setLikeCount(post.getLikeCount() + 1);
                    post.setDislikeCount(Math.max(0, post.getDislikeCount() - 1)); }
                else { post.setDislikeCount(post.getDislikeCount() + 1);
                    post.setLikeCount(Math.max(0, post.getLikeCount() - 1)); }
                ev.setVoteType(voteType); voteRepository.save(ev);
            }
        } else {
            // ③ 无记录：新增
            ForumVote newVote = new ForumVote();
            newVote.setPostId(postId); newVote.setUserId(userId);
            newVote.setUserRole(userRole); newVote.setVoteType(voteType);
            voteRepository.save(newVote);
            if ("like".equals(voteType)) post.setLikeCount(post.getLikeCount() + 1);
            else post.setDislikeCount(post.getDislikeCount() + 1);
        }
        postRepository.save(post);
        return toVO(post, userId, userRole, false);
    }
}
```

帖子列表的查询逻辑分为两条路径：无搜索时分别查询置顶帖列表与普通帖分页，首页将两者合并使置顶帖始终位于列表最前；有关键词时直接按标题模糊匹配分页返回。投票操作按三段式逻辑分发——同类型取消、异类型切换、无记录新增——完整覆盖用户的全部投票操作路径。

**3. 控制器层——帖子管理接口**

```java
// ForumController.java - 帖子管理的 REST 接口
@RestController
@RequestMapping("/forum")
public class ForumController {
    @Autowired
    private ForumService forumService;

    // 获取帖子列表（搜索、排序、分页）
    @GetMapping("/posts")
    public ResponseResult<Page<ForumPostVO>> getPosts(
            @RequestParam(required = false) String keyword,
            @RequestParam(defaultValue = "newest") String sort,
            @RequestParam(defaultValue = "0") int page,
            @RequestParam(defaultValue = "10") int size,
            @AuthenticationPrincipal AuthenticatedUser currentUser) {
        return ResponseResult.success(forumService.getPosts(
                keyword, sort, page, size,
                currentUser != null ? currentUser.getUserId() : null,
                currentUser != null ? currentUser.getRoleValue() : null));
    }

    // 发帖（需登录）
    @PreAuthorize("isAuthenticated()")
    @PostMapping("/posts")
    public ResponseResult<ForumPost> createPost(
            @RequestBody Map<String, Object> body,
            @AuthenticationPrincipal AuthenticatedUser currentUser) {
        return ResponseResult.success(forumService.createPost(
                (String) body.get("title"), (String) body.get("content"),
                currentUser.getUserId(), currentUser.getDisplayName(), currentUser.getRoleValue()));
    }

    // 点赞/踩（需登录）
    @PreAuthorize("isAuthenticated()")
    @PostMapping("/posts/{id}/vote")
    public ResponseResult<ForumPostVO> vote(@PathVariable Long id,
            @RequestBody Map<String, Object> body,
            @AuthenticationPrincipal AuthenticatedUser currentUser) {
        return ResponseResult.success(forumService.vote(
                id, currentUser.getUserId(), currentUser.getRoleValue(), (String) body.get("voteType")));
    }

    // 置顶/取消置顶（仅管理员）、删除帖子（管理员或作者）略...
}
```

帖子管理接口采用分层权限控制：发帖、删除与投票要求已登录，置顶进一步限定管理员角色。列表接口为开放访问，未登录用户亦可浏览论坛内容。用户发帖时，前端将标题与内容封装为 JSON 对象通过 `POST /forum/posts` 发送至后端，控制器通过 `@AuthenticationPrincipal` 从 JWT 令牌中提取当前用户信息后传入服务层，服务层将 title、content、authorId、authorName、authorRole 五个字段写入 ForumPost 实体并持久化。

### 4.5.2 评论管理模块

评论功能使帖子具备了多轮异步对话的能力，每条帖子可承载多条评论，按时间正序排列。用户可在帖子详情页查看全部评论并发表新评论，管理员有权删除不当评论。

**1. 实体层——ForumComment 实体**

```java
// ForumComment.java - 论坛评论实体，映射数据库 forum_comment 表
@Entity
@Table(name = "forum_comment")
public class ForumComment {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private Long postId;              // 所属帖子 ID
    private Long authorId;            // 评论者 ID
    private String authorName;        // 冗余存储，避免联表查询
    private String authorRole;        // 评论者角色：student / teacher / admin
    @Column(columnDefinition = "TEXT")
    private String content;           // 评论内容（长文本）
    private LocalDateTime createdAt;  // 创建时间，@PrePersist 自动填充
    // getter / setter 略...
}
```

ForumComment 实体中 authorName 与 authorRole 为冗余字段，使评论列表渲染时无需对每条评论执行用户表的联表查询，显著降低了列表页的数据库负载。

**2. 服务层——发表评论**

```java
// ForumService.java - 评论发布与帖子详情方法
@Service
public class ForumService {
    @Autowired
    private ForumCommentRepository commentRepository;  // 评论 JPA 数据访问接口

    // 发表评论：校验帖子存在后构建实体并写入数据库
    public ForumComment addComment(Long postId, Long authorId, String authorName,
                                   String authorRole, String content) {
        if (!postRepository.existsById(postId))
            throw new RuntimeException("帖子不存在");
        ForumComment comment = new ForumComment();
        comment.setPostId(postId); comment.setAuthorId(authorId);
        comment.setAuthorName(authorName); comment.setAuthorRole(authorRole);
        comment.setContent(content);
        return commentRepository.save(comment);  // JPA save 方法持久化
    }
}
```

**3. 控制器层——评论管理接口**

```java
// ForumController.java - 帖子详情与评论管理的 REST 接口
@RestController
@RequestMapping("/forum")
public class ForumController {
    // 获取帖子详情（含评论列表与当前用户投票状态）
    @GetMapping("/posts/{id}")
    public ResponseResult<ForumPostVO> getPostDetail(@PathVariable Long id,
            @AuthenticationPrincipal AuthenticatedUser currentUser) {
        return ResponseResult.success(forumService.getPostDetail(
                id, currentUser.getUserId(), currentUser.getRoleValue()));
    }

    // 发表评论（需登录）
    @PreAuthorize("isAuthenticated()")
    @PostMapping("/posts/{id}/comments")
    public ResponseResult<ForumComment> addComment(@PathVariable Long id,
            @RequestBody Map<String, Object> body,
            @AuthenticationPrincipal AuthenticatedUser currentUser) {
        return ResponseResult.success(forumService.addComment(
                id, currentUser.getUserId(), currentUser.getDisplayName(),
                currentUser.getRoleValue(), (String) body.get("content")));
    }
}
```

评论管理接口的权限设计简洁明确：发表评论仅需登录认证，删除评论限定管理员角色。帖子详情接口返回的 ForumPostVO 中包含 comments 列表与 myVote 标识，前端据此渲染评论区与当前用户的投票状态。

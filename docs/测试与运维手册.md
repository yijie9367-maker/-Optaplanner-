## 三、API 接口测试

### 3.1 接口清单

| 模块 | 方法 | 路径 | 需认证 |
|------|------|------|--------|
| 登录 | POST | `/student/login` | ❌ |
| 课表 | GET | `/schedule/listByClassGroupId?classGroupId=1` | ✅ |
| 考试 | GET | `/exam/listByClassGroupId?classGroupId=1` | ✅ |
| 论坛列表 | GET | `/forum/posts?sort=newest&page=0&size=10` | ❌ |
| 论坛详情 | GET | `/forum/posts/{id}` | ❌ |
| 发帖 | POST | `/forum/posts` | ✅ |
| 点赞 | POST | `/forum/posts/{id}/vote` | ✅ |
| 评论 | POST | `/forum/posts/{id}/comments` | ✅ |
| 学生信息 | GET | `/student/{id}` | ✅ |
| 更新资料 | PUT | `/student/update` | ✅ |
| AI 对话 | POST | `/chatbot/chat` | ❌ |
| 匿名配置 | GET | `/chatbot/anonymous-config` | ❌ |

### 3.2 测试命令（curl / Postman）

```bash
# 1. 学生登录
curl -X POST http://10.17.86.214:8084/student/login \
  -H "Content-Type: application/json" \
  -d '{"name":"张三","password":"123456"}'
# 预期: {"code":200,"data":{"token":"eyJ...","student":{...}}}

# 2. 获取课表（替换 TOKEN）
curl http://10.17.86.214:8084/schedule/listByClassGroupId?classGroupId=1 \
  -H "Authorization: Bearer {TOKEN}"
# 预期: {"code":200,"data":[{"courseName":"高等数学",...}]}

# 3. 获取考试安排
curl http://10.17.86.214:8084/exam/listByClassGroupId?classGroupId=1 \
  -H "Authorization: Bearer {TOKEN}"

# 4. 论坛帖子列表
curl "http://10.17.86.214:8084/forum/posts?sort=newest&page=0&size=10"

# 5. 论坛帖子详情
curl http://10.17.86.214:8084/forum/posts/1

# 6. 发帖
curl -X POST http://10.17.86.214:8084/forum/posts \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer {TOKEN}" \
  -d '{"title":"测试标题","content":"测试内容","authorId":1,"authorName":"张三","authorRole":"student"}'

# 7. 点赞
curl -X POST http://10.17.86.214:8084/forum/posts/1/vote \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer {TOKEN}" \
  -d '{"userId":1,"userRole":"student","voteType":"upvote"}'

# 8. 发评论
curl -X POST http://10.17.86.214:8084/forum/posts/1/comments \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer {TOKEN}" \
  -d '{"authorId":1,"authorName":"张三","authorRole":"student","content":"好帖"}'

# 9. AI 聊天
curl -X POST http://10.17.86.214:8084/chatbot/chat \
  -H "Content-Type: application/json" \
  -d '{"message":"你好","sessionId":""}'
# 预期: {"code":200,"data":{"reply":"你好！👋 我是AI智能排课助手...","sessionId":"xxx"}}

# 10. AI 匿名配置
curl http://10.17.86.214:8084/chatbot/anonymous-config
```

### 5.2 配置文件管理

#### `application.yml` 关键配置

```yaml

  port: 8084                          # 后端端口

  datasource:
    url: jdbc:mysql://localhost:3306/aischedule?...
    username: root                     # 生产环境勿用 root
    password: 123456                   # 生产环境加密

app:
  jwt:
    secret: AISCHEDULE_JWT_SECRET_KEY_2026_WJ_PROJECT_SECURE_32
    expiration-minutes: 720            # Token 12小时过期

appflow:
  chat:
    integrate-id: "cit-xxx"            # 百炼应用ID
    request-domain: "https://xxx.appflow.aliyunnest.com"
    api-key: "sk-xxx"                  # API密钥（勿提交到公开仓库）
    api-path: "/webhook/chatbot/chat/"
```

### 5.6 域名与 HTTPS

| 配置项 | 位置 | 说明 |
|------|------|------|
| 业务域名 | 微信公众平台 → 开发管理 → 开发设置 | web-view 加载的域名需加入白名单 |
| request 合法域名 | 微信公众平台 → 开发管理 → 开发设置 | `wx.request` 的域名需 HTTPS |
| SSL 证书 | Nginx / 后端 | 生产环境必须 HTTPS |

Nginx 反向代理示例：

```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;

    ssl_certificate     /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://127.0.0.1:8084;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 六、常见故障排查

### 6.1 后端启动失败

| 现象 | 排查 |
|------|------|
| 端口被占用 | `netstat -ano \| findstr :8084`，换端口或杀掉占用进程 |
| 数据库连接失败 | 检查 MySQL 是否运行，`url`/`username`/`password` 是否正确 |
| JWT 密钥长度不足 | `app.jwt.secret` 至少 32 字节 |
| 表不存在 | 执行 `schema.sql` 初始化表结构 |

### 6.2 小程序请求失败

| 现象 | 排查 |
|------|------|
| `request:fail` | ① 后端是否启动 ② IP 是否正确 ③ 是否勾选「不校验合法域名」|
| 401 未授权 | Token 过期，重新登录 |
| 500 服务器错误 | 查看后端控制台日志 |
| web-view 加载失败 | ① `chatbot.html` 是否存在 ② 业务域名是否配置 ③ 是否勾选「不校验 web-view 域名」|

### 6.3 AI 助手不回复

| 现象 | 排查 |
|------|------|
| SDK 不加载 | 检查 `integrateId`/`requestDomain` 是否正确 |
| 总是模拟回复 | `application.yml` 中 `appflow.chat` 未正确配置或百炼 API 不可达 |
| ACCOUNT_NOT_LOGIN | 百炼 API 需登录认证，当前后端直调不支持，请使用 SDK 方式 |
| 404 Not Found | `api-path` 不正确 |

### 6.4 课表/考试无数据

| 现象 | 排查 |
|------|------|
| 未登录 | 先登录获取 classGroupId |
| classGroupId 不对 | 检查学生绑定的班级ID是否有课表数据 |
| 当前周次无排课 | 切换到其他周次 |
| 数据库无数据 | 导入排课/考试数据 |

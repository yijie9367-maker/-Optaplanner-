# AISchedule 更新日志 — 本次会话变更汇总

## 一、AI 回复换行修复

**问题**：后端返回的 AI 文本含 `\n`，前端显示时全部粘在一起。

**原因**：微信小程序中 `<view>` 不识别 `\n`，只有 `<text>` 组件才会渲染换行。

**修改**：
| 项目 | 文件 | 改动 |
|------|------|------|
| 小程序 | `components/ai-chat/ai-chat.wxml` | 气泡内用 `<text>` 包裹 `{{item.text}}` |
| 小程序 | `components/ai-chat/ai-chat.wxss` | 加 `.bub text { white-space: pre-wrap; }` |

```xml
<!-- 改前 -->
<view class="bub bub-left">{{item.text}}</view>
<!-- 改后 -->
<view class="bub bub-left"><text>{{item.text}}</text></view>
```

## 八、排课冲突检测

**问题**：手动添加排课时，同一位老师在同一时段可被安排多门课，无任何提示。

**修改**：
| 文件 | 改动 |
|------|------|
| `repository/ScheduleRepository.java` | 新增 `findByWeekDayAndWeekNumber()` |
| `service/ScheduleService.java` | 新增 `checkConflict()` + `timeSlotsOverlap()`，save 前检测 |
| `controller/ScheduleController.java` | `add()`/`update()` 加 try-catch 返回错误 |

**检测维度**：教师冲突 / 教室冲突 / 班级冲突（同周次+同星期+时段重叠）

**错误示例**：
```
教师冲突：张老师 在 2024-spring 第3周 周一 1-2节 已有课程安排，无法重复安排
教室冲突：A201 在 2024-spring 第3周 周一 1-2节 已被占用，无法重复安排
```

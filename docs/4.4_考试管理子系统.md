## 4.4 考试管理子系统详细设计与实现

排课完成后，教学计划进入执行阶段，考试安排与成绩管理成为衔接教学与评价的关键环节。考试管理子系统提供两类核心功能：其一为考试安排管理，管理员可为各班级、各课程统一编排考试时间与地点，授课教师亦可对所授课程的考试进行调整；其二为考试成绩管理，教师完成阅卷后可批量录入或导入学生成绩，学生登录系统后即可查看个人各科成绩。本节将分别阐述考试安排管理模块与考试成绩管理模块的设计与实现。

### 4.4.1 考试安排管理模块

考试安排管理模块为管理员和教师提供考试信息的创建、查询、修改与删除功能。考试安排关联课程、班级和授课教师三个维度，授课教师由课程信息自动关联。管理员具有全局操作权限，可管理所有考试安排；教师仅能操作其所授课程的考试，以此保证数据权限的合理隔离。

后端基于 Spring Boot + JPA 构建 RESTful 接口，通过 `@PreAuthorize` 注解实现角色级校验，同时在服务层注入请求者身份以完成数据级权限隔离。考试安排管理运行效果图如图 4.X 所示。

**图 4.X 考试安排管理运行效果图**

考试安排涉及课程、班级、教师三个核心数据维度，实体设计上以三个外键字段关联对应的基础信息表。其中 `teacherId` 不由前端传入，而是在服务层根据课程的授课教师自动回填，避免用户手动选择时可能出现的数据不一致。

**1. 实体层——Exam 实体**

```java
// Exam.java - 考试安排实体，映射数据库 exam 表
@Entity
@Table(name = "exam")
public class Exam {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private Long courseId;        // 课程ID
    private Long classGroupId;    // 班级ID
    private Long teacherId;       // 教师ID（由课程授课教师自动关联）
    private LocalDate examDate;   // 考试日期
    private String startTime;     // 开始时间
    private String endTime;       // 结束时间
    private String location;      // 考场地点
    // getter / setter 略...
}
```

考试安排创建时涉及多重业务约束，不能简单地将前端数据直接入库。服务层的 `save` 方法承担了全部前置校验职责，按优先级依次执行字段完整性、时间逻辑、关联数据存在性、操作权限及唯一性共五层校验，构成一套完整的业务规则链。

**2. 服务层——考试安排的业务校验**

```java
// ExamService.java - save方法：创建或更新考试安排，含完整业务校验
@Service
public class ExamService {

    @Autowired
    private ExamRepository examRepository;
    @Autowired
    private CourseService courseService;
    @Autowired
    private ClassGroupService classGroupService;

    public Exam save(Exam exam, String requesterRole, Long requesterId) {
        // ① 字段非空校验
        if (exam.getCourseId() == null) throw new RuntimeException("请选择课程");
        if (exam.getClassGroupId() == null) throw new RuntimeException("请选择班级");
        if (exam.getExamDate() == null) throw new RuntimeException("请选择考试日期");

        // ② 时间合法性校验
        if (exam.getStartTime().compareTo(exam.getEndTime()) >= 0)
            throw new RuntimeException("结束时间必须晚于开始时间");

        // ③ 关联数据存在性校验 & 授课教师自动关联
        Course course = courseService.findById(exam.getCourseId());
        if (course == null) throw new RuntimeException("课程不存在");
        exam.setTeacherId(course.getTeacherId());

        // ④ 教师数据权限校验：教师只能创建自己授课课程的考试
        if ("teacher".equals(requesterRole) && !course.getTeacherId().equals(requesterId))
            throw new RuntimeException("教师只能为自己授课的课程创建或修改考试");

        // ⑤ 唯一性约束：同一班级的同一门课程只能有一场考试
        Exam existing = examRepository
                .findByCourseIdAndClassGroupId(exam.getCourseId(), exam.getClassGroupId())
                .orElse(null);
        if (existing != null && !existing.getId().equals(exam.getId()))
            throw new RuntimeException("该班级的这门课程考试已存在");

        return examRepository.save(exam);
    }
}
```

上述 `save` 方法体现了业务层在考试安排创建中的核心价值——并非简单地将数据写入数据库，而是依次执行校验并以明确的异常信息终止不合规操作。五层校验中任一环节失败均立即抛出异常，确保数据的一致性与安全性。

控制器层接收前端请求后，将业务逻辑全部委托给服务层处理。为便于前端表格展示课程名称、班级名称与教师姓名等关联信息，服务层在查询时通过 `ExamVO` 视图对象完成 Entity 到 VO 的转换，将外键 ID 替换为关联表的名称字段，降低前端二次拼接数据的复杂度。

**3. 控制器层——考试安排管理接口**

```java
// ExamController.java - 考试安排管理的 RESTful 接口
@RestController
@RequestMapping("/exam")
public class ExamController {

    @Autowired
    private ExamService examService;

    // 查询全部考试安排
    @PreAuthorize("hasAnyRole('ADMIN','TEACHER')")
    @GetMapping("/list")
    public ResponseResult<List<ExamVO>> list() {
        return ResponseResult.success(examService.findAllVO());
    }

    // 新增或更新考试安排
    @PreAuthorize("hasAnyRole('ADMIN','TEACHER')")
    @PostMapping("/save")
    public ResponseResult<Exam> save(@RequestBody Exam exam,
            @RequestAttribute String role, @RequestAttribute Long userId) {
        return ResponseResult.success(examService.save(exam, role, userId));
    }

    // 删除考试安排
    @PreAuthorize("hasAnyRole('ADMIN','TEACHER')")
    @DeleteMapping("/delete/{id}")
    public ResponseResult<Void> delete(@PathVariable Long id,
            @RequestAttribute String role, @RequestAttribute Long userId) {
        examService.deleteById(id, role, userId);
        return ResponseResult.success();
    }
}
```

管理员或教师登录后，前端调用 `GET /exam/list` 接口获取全部考试安排并以表格形式展示。接口通过 `@PreAuthorize` 进行角色级权限校验，确保仅 ADMIN 和 TEACHER 角色可访问。在新增或删除考试时，控制器从请求上下文中提取当前用户角色与 ID 并传入服务层，由服务层实现数据级权限隔离——教师只能操作其所授课程的考试，管理员则无此限制。

### 4.4.2 考试成绩管理模块

考试安排就绪后，教师需在考试结束后为学生录入成绩。考试成绩管理模块提供成绩录入、批量导入导出以及学生成绩查询功能。成绩实体以考试 ID 与学生 ID 建立复合唯一约束，确保同一场考试中每位学生仅有一条成绩记录，避免重复录入。

**1. 实体层——ExamGrade 实体**

```java
// ExamGrade.java - 考试成绩实体，映射 exam_grade 表
@Entity
@Table(name = "exam_grade",
       uniqueConstraints = @UniqueConstraint(columnNames = {"exam_id", "student_id"}))
public class ExamGrade {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    @Column(name = "exam_id", nullable = false)
    private Long examId;              // 考试ID
    @Column(name = "student_id", nullable = false)
    private Long studentId;           // 学生ID
    @Column(name = "score", precision = 5, scale = 2)
    private BigDecimal score;         // 分数
    // getter / setter 略...
}
```

ExamGrade 实体在表级别通过 `@UniqueConstraint` 声明了考试 ID 与学生 ID 的复合唯一约束，从数据库层面杜绝了同一学生在同一场考试中出现多条成绩记录的可能，配合服务层校验逻辑构成双重保障。

成绩录入场景中，教师需要看到该班级的完整学生名单，而非仅已有成绩的学生。为此，服务层采用"先查全班学生，再左连接已有成绩"的策略，将班级学生列表与成绩记录按学生 ID 做映射，已有分数的显示数值，尚未录入的保留空值。

**2. 服务层——成绩查询与批量保存**

```java
// ExamGradeService.java - 成绩管理核心业务方法
@Service
public class ExamGradeService {

    @Autowired
    private ExamGradeRepository examGradeRepository;
    @Autowired
    private StudentRepository studentRepository;

    // 教师：查询某场考试下某班级所有学生的成绩（含未录入学生）
    public List<GradeItem> findStudentsWithGradeByExamAndClass(Long examId, Long classGroupId) {
        List<Student> students = studentRepository.findByClassGroupId(classGroupId);
        Map<Long, ExamGrade> gradeMap = examGradeRepository.findByExamId(examId)
                .stream().collect(Collectors.toMap(ExamGrade::getStudentId, g -> g));

        return students.stream().map(student -> {
            GradeItem item = new GradeItem();
            item.setStudentId(student.getId());
            item.setStudentName(student.getName());
            ExamGrade grade = gradeMap.get(student.getId());
            if (grade != null) item.setScore(grade.getScore());
            return item;
        }).collect(Collectors.toList());
    }

    // 批量保存一场考试的成绩
    @Transactional
    public void saveAll(Long examId, List<GradeItem> items) {
        for (GradeItem item : items) {
            ExamGrade grade = examGradeRepository
                    .findByExamIdAndStudentId(examId, item.getStudentId())
                    .orElse(new ExamGrade());
            grade.setExamId(examId);
            grade.setStudentId(item.getStudentId());
            grade.setScore(item.getScore());
            examGradeRepository.save(grade);
        }
    }
}
```

`findStudentsWithGradeByExamAndClass` 方法是成绩管理模块的核心逻辑。教师选择某场考试和对应班级后，后端查得该班级全部学生列表及已有成绩记录，以学生 ID 为键将成绩映射至 GradeItem 对象。`saveAll` 方法则采用"先查询后更新"的策略——对每条 GradeItem 先查询是否已有成绩记录，已有则更新分数，无则新建，并通过 `@Transactional` 注解保证批量操作的原子性。

**3. 控制器层——成绩管理接口**

```java
// ExamGradeController.java - 成绩查询与录入接口
@RestController
@RequestMapping("/exam-grade")
public class ExamGradeController {

    @Autowired
    private ExamGradeService examGradeService;

    // 教师：查询某场考试某班级所有学生的成绩
    @PreAuthorize("hasAnyRole('ADMIN','TEACHER')")
    @GetMapping("/listByExamAndClass")
    public ResponseResult<List<GradeItem>> listByExamAndClass(
            @RequestParam Long examId, @RequestParam Long classGroupId) {
        return ResponseResult.success(
                examGradeService.findStudentsWithGradeByExamAndClass(examId, classGroupId));
    }

    // 学生：查询自己所有的考试成绩
    @PreAuthorize("@authz.isStudentOrAdmin(#studentId)")
    @GetMapping("/listByStudentId")
    public ResponseResult<List<ExamGrade>> listByStudentId(@RequestParam Long studentId) {
        return ResponseResult.success(examGradeService.findByStudentId(studentId));
    }

    // 教师：批量保存一场考试的成绩
    @PreAuthorize("hasAnyRole('ADMIN','TEACHER')")
    @PostMapping("/saveAll")
    public ResponseResult<Void> saveAll(@RequestBody Map<String, Object> body) {
        Long examId = Long.valueOf(body.get("examId").toString());
        List<GradeItem> items = parseGradeItems(body);
        examGradeService.saveAll(examId, items);
        return ResponseResult.success();
    }
}
```

成绩管理接口的权限控制采用双重策略。教师端的查询与录入接口使用 `@PreAuthorize("hasAnyRole('ADMIN','TEACHER')")`，允许管理员与教师角色访问；学生端的个人成绩查询则通过自定义权限表达式 `@authz.isStudentOrAdmin(#studentId)` 进行细粒度控制——仅允许学生查看本人的成绩，或管理员查看任意学生的成绩。此外，为方便管理员进行批量数据处理，成绩管理模块额外提供了 Excel 导入导出功能，管理员可下载模板批量导入成绩或一键导出全部数据用于归档分析。

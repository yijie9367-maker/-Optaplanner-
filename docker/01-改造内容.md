# 01 改造内容

## 一、为什么要改

改造前，这个项目的登录和权限控制更偏“演示型”：

- 管理员、教师、学生登录直接在控制器里写校验逻辑
- 返回的是字符串拼接的伪 token，例如 `student-token-xxx`
- 多个接口通过判断 `Authorization` 是否以 `admin-token-` 开头来决定权限
- 密码基本按明文比较

这类实现可以跑通页面，但技术含量和安全性都不够强，不适合当成一个成熟的毕业设计后端方案。

## 二、这次具体改了什么

### 1. 后端依赖升级

在 `aischedule/pom.xml` 中新增：

- `spring-boot-starter-security`
- `jjwt-api`
- `jjwt-impl`
- `jjwt-jackson`

这意味着项目正式具备：

- Spring Security 无状态鉴权能力
- JWT 令牌签发与解析能力

### 2. 新增安全基础设施

在后端新增了安全相关模块：

- `security/AppRole.java`
- `security/AuthenticatedUser.java`
- `security/PasswordCodecService.java`
- `security/JwtTokenService.java`
- `security/JwtAuthenticationFilter.java`
- `security/RestAuthenticationEntryPoint.java`
- `security/RestAccessDeniedHandler.java`
- `security/SecurityAccessService.java`
- `config/SecurityConfig.java`

这些类分别负责：

- 统一角色枚举
- 统一当前登录用户对象
- 密码加密与兼容校验
- JWT 签发与解析
- 请求过滤器中自动识别 Bearer Token
- 未登录 / 无权限 时返回统一 JSON
- 在接口层做角色判断
- 注册整套 Spring Security 配置

### 3. 登录逻辑统一

新增了 `service/AuthService.java`，把管理员、教师、学生三套登录逻辑统一起来。

目前保留原有登录入口，兼容现有前端：

- `/admin/login`
- `/teacher/login`
- `/student/login`
- `/login`（学生端实际使用）

但现在登录成功后返回的已经是**真实 JWT**，不再是拼接字符串。

### 4. 密码机制升级

原来的密码比较基本是明文。

现在改成：

- 新增 / 更新用户时，自动使用 `BCrypt` 加密密码
- 历史明文密码仍可兼容登录
- 历史明文密码在登录成功后会自动迁移为加密密码
- 教师 / 学生若历史密码为空，兼容默认密码 `123456`

这样既避免一次性把老数据全部打坏，又把系统逐步迁移到了安全存储模式。

### 5. 接口权限控制升级

原来很多接口靠控制器里手写判断请求头前缀。

现在改成：

- 通过 Spring Security 统一识别 JWT
- 通过 `@PreAuthorize` 控制接口角色
- 通过 `@authz.xxx(...)` 做“当前用户是否有权访问这条数据”的校验

例如：

- 管理员接口：`hasRole('ADMIN')`
- 教师查看自己的考试：`@authz.isTeacherOrAdmin(#teacherId)`
- 学生查看自己班级课表：`@authz.isStudentClassOwnerOrAdmin(#classGroupId)`

### 6. 论坛接口不再信任前端传来的作者身份

改造前，论坛发帖、评论、点赞时，后端直接信任前端 body 中的：

- `authorId`
- `authorName`
- `authorRole`

这意味着理论上可以伪造身份。

改造后：

- 后端优先从 JWT 中取当前登录用户
- 发帖 / 评论 / 点赞的作者信息由后端从登录态确定
- 删除帖子时，会校验“是否管理员或帖子作者”

### 7. 前端同步修正了 fetch 调用

教师端、学生端、管理员考试页里有一部分接口原来直接使用 `fetch`，没有自动附带 token。

这次新增了：

- `aischedule-frontend/src/utils/authFetch.js`

并把相关页面改成通过 `authFetch` 自动带上 `Authorization: Bearer <token>`。

这样 JWT 鉴权启用后，教师/学生端页面不会因为请求没带 token 而失效。

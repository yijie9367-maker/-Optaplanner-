# 04 联调与使用说明

## 一、登录接口

### 管理员登录

```http
POST /admin/login
Content-Type: application/json

{
  "username": "admin",
  "password": "123456"
}
```

### 教师登录

```http
POST /teacher/login
Content-Type: application/json

{
  "name": "张老师",
  "password": "123456"
}
```

### 学生登录

```http
POST /login
Content-Type: application/json

{
  "name": "王同学",
  "password": "123456"
}
```

## 二、登录成功后的返回

响应里会包含：

- 用户基本信息
- `role`
- `token`

其中 `token` 就是之后访问接口需要携带的 JWT。

## 三、前端如何带 token

请求头格式：

```http
Authorization: Bearer <token>
```

本次已经新增：

```text
aischedule-frontend/src/utils/authFetch.js
```

用于把 localStorage 中的 token 自动加到 `fetch` 请求头里。

## 四、哪些接口已经纳入权限控制

### 管理员接口

例如：

- 管理员账号管理
- 学生/教师/班级/教室/课程管理
- 约束配置管理
- 智能排课执行
- 公告管理

### 教师接口

例如：

- 查看自己的课表
- 查看自己的考试
- 录入成绩

### 学生接口

例如：

- 查看自己班级的课表
- 查看自己班级的考试
- 查看自己的成绩

## 五、当前密码兼容规则

### 管理员

- 老明文密码仍可登录
- 登录成功后会迁移为 BCrypt

### 教师 / 学生

- 老明文密码仍可登录
- 如果历史密码为空，兼容默认密码 `123456`
- 登录成功后会迁移为 BCrypt

## 六、你后续如果还想继续做的增强

如果你想继续把后端安全这块再往上做，可以继续扩展：

1. 刷新令牌（Refresh Token）
2. 修改密码接口
3. 登录日志 / 操作审计日志
4. 黑名单令牌或 Redis 失效控制
5. 更细粒度的菜单权限 / 按钮权限

## 七、你自己复盘时可以重点记住的三个概念

### 1. 认证 Authentication

系统确认“你是谁”。

### 2. 授权 Authorization

系统确认“你能做什么”。

### 3. 鉴权上下文 Security Context

系统在请求处理过程中，把当前用户身份放到上下文里，后面的控制器和权限表达式都从这里读取。
